3 Business · users · use-cases

Business. The neutral, non-custodial control tower for AI agents that spend. It rides the payment rails (x402 / settlement) and owns the layer rivals treat as an afterthought — policy, human oversight, operator UX. Non-custodial is locked (sidesteps money-transmitter licensing); monetization leans to a credits/SaaS model.

Users

  • Beachhead: agent-payments builders who just realized "this agent can drain my wallet."
  • Economic buyer: whoever gets blamed if an agent overspends — finance/ops, the solo founder, the agency owner.
  • Expansion: non-crypto "agent ops" operators who want a dashboard, not an SDK.

Use-cases (most real first)

  • Agent buys paid data / APIs per call (the demo — canonical x402).
  • Autonomous research / procurement on a fixed budget.
  • Agent buys compute/inference metered against a cap.
  • Agent-to-agent micropayments (emerging).

Job-to-be-done: "Let an AI agent do work that costs money — without giving it unbounded access to my money."

4 The real-case scenario to demo

"You're an analyst. You delegate 'compare AI coding tools 2026' to an agent with a 50 XRP budget — auto-pay under 0.1 XRP, block over 0.2. Watch it buy the data, settle on-chain, and hand you a sourced report — and prove you're in control."
1
The happy path. Money=Live, Agent=Live, paste an OpenRouter key, pick a model, band 0.1 / 0.2. Launch → feed streams probe → policy allow → real settled tx → unlocked → AI synthesis → complete. Click View on Explorer. Proves the whole loop + on-chain proof.
2
Human in the loop. Set min = 0 (everything needs approval) → launch → Approve/Deny modal → Approve → settles. Re-run → Deny → refused, no payment. Proves gate 6 is tactile and real.
3
Kill switch. Launch, hit 🛑 KILL mid-run → policy refuses, no signature. Proves the emergency control.
4
The payoff. Show the Result panel (report + cost breakdown + fee) and Export. Proves the deliverable + transparency.

5 Acceptance checklist — announce when all green

  • Policy engine has automated tests; npm test green (23 tests)
  • Deploy config ready (Dockerfile + Render blueprint; PORT-aware)
  • Deployed to a public URL, loads clean, wallets pre-funded
  • Fresh visitor completes scenario step 1 (real tx + explorer) in <60s
  • Real AI synthesis renders with a real OpenRouter key (not just routing)
  • Approve, Deny, and Kill each demonstrably change the on-chain outcome
  • Graceful empty/error states (wallet dry, merchant down, bad key)
  • 60–90s demo video recorded
  • README "run in 2 minutes" + the public link
Shortest path to announceable: (1) one real-key AI run → (2) ✅ policy tests → (3) push the Docker/Render deploy to a public URL → (4) record the 90-second video. Items 1, 3-deploy-click, and 4 need a human (a key, a hosting account, a screen recorder); everything else is done.
