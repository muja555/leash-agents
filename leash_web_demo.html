Leash — the web demo

Open a URL. Paste a key. Launch an agent. Watch a real testnet payment settle in your browser.

Leash v1 ships as a web app at localhost:8080 — vanilla HTML/CSS/JS in front, Express + Server-Sent Events behind. BYOK Anthropic key in localStorage means the secret never leaves your browser. Same M1 loop on the backend; only the shell changes.

01

Setup · 3 steps

From clone to first testnet payment in two minutes.

Install + configure

npm install
cp .env.example .env

First run leaves the wallet seeds blank; the server auto-funds a merchant and an agent wallet from the testnet faucet and prints the seeds so you can save them back into .env for re-runs.

Boot the web app

npm run web

Boots Express + SSE backend + serves public/index.html as the frontend on http://localhost:8080. The same Express instance also serves the M1 merchant routes (/research, /api/policy, /api/task).

Open the page

Open http://localhost:8080 in any browser. Type a query, click ▶ Launch Agent. The live feed pane streams every loop step — the green TX SETTLED bubble has a real explorer link.

03

Security model — BYOK in the browser

Your Anthropic key is a high-trust secret. Here's exactly what Leash does with it, and what it never does.

WHAT LEASH DOES

Stored in: your browser's localStorage under the key leash.anthropic_key. Same security model as any other JS app's localStorage — visible to JavaScript running on the page, not visible to other sites.

Sent to backend: in the JSON body of POST /api/task over the SSE stream, only when you click Launch Agent.

Used in backend: held in memory for the duration of the task, passed to the Anthropic SDK for that task only.

Forgotten: when the SSE stream closes. No file, no database, no log.

RECOMMENDED PRACTICE

① Create a NEW key in the Anthropic Console specifically for Leash — don't reuse your main key.

② Set a hard spend cap on it ($5–$20/mo). Anthropic Console supports per-key budgets.

③ Revoke from the console if anything feels off. The cap bounds your worst case.

04

What this demo does NOT do yet

Honest about M2 scope so you can read the demo correctly. None of these are blockers; they're the M3 punch list.

M3 PUNCH LIST

① Claude reasoning — the agent is still deterministic in M2. The BYOK key is captured in the UX but the Anthropic API isn't called yet. M3 wires the tool-use loop. M3

② Approve / Deny modal — every payment is below the threshold in M2, so the prompt never fires. M3 shows a web modal when a payment crosses the threshold; you click to release or refuse the signature. M3

③ Per-user wallet + deposit flow — M2 uses a shared agent wallet auto-funded from the testnet faucet. M3 generates a per-user wallet, shows the address, and links to the testnet faucet so visitors can fund their own agent. M3

④ Hard kill switch — the KILL button blocks new tasks but doesn't interrupt an in-flight run. M3 routes the halt flag through the policy engine, refusing every future signature mid-run. M3
